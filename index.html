Welcome to the IWS

IWS, IntraWeb Services, was originally created by Kim Jensen for IAESTE. But to ensure a broader exposure for other IAESTE Committees around the world, who would like to interact with it, it was published as an Open Source Project in February 2016.

What it is

IWS is the back-end system for the IAESTE IntraWeb. It is written in Java as a pure JaveEE Project, originally running under Glassfish, but later moved to the Open Source JBoss - WildFly. It is build using Maven, and requires Java7 or later. The Source Code is kept at source level 7 to be compatible with the current front-end, which is not published as Open Source.

Who is this for

It is primarily intended for IAESTE A.s.b.l. But it is made readily available to anyone who wishes to either build their own interaction tools for the IWS or use it as a base for their own systems.

Which release should I choose

Currently, the IWS is undergoing a bit of code restructuring to get rid of the old "Proto Type" feel. Loads of things were hacked into it under heavy time constraints, and many things have evolved since it was originally written. To help finalize this part, a mixture of monitoring systems is used, including Sonar, Codacy & Coverity. The goal is that all 3 systems will give a clean report, and that all currently open issues for version 1.2 is completed.

In the meantime, please use the current snapshot and build based on it, the API is almost completely locked and the testing with IW4 has shown that everything works.
